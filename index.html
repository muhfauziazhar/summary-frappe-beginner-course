1. Pendahuluan & Konsep Dasar

Apa itu Frappe Framework & ERPNext?

Frappe Framework adalah platform "batteries-included" yang memungkinkan pengembangan aplikasi web berbasis database dengan cepat. Dibangun dengan Python & JavaScript, ia menyediakan fitur-fitur siap pakai seperti ORM, sistem otentikasi, UI admin, dan banyak lagi. ERPNext adalah produk unggulan yang dibangun di atas Frappe, sebuah solusi ERP open-source yang lengkap untuk manajemen bisnis.

Struktur Direktori

Memahami struktur direktori adalah kunci. Saat Anda menginisialisasi Frappe Bench, direktori berikut akan dibuat. Setiap aplikasi yang Anda buat atau instal akan memiliki struktur internalnya sendiri, termasuk folder untuk modul, doctype, halaman, dan laporan.

  • /apps: Tempat semua aplikasi (termasuk Frappe & ERPNext) berada.
  • /sites: Berisi semua situs yang Anda buat, termasuk file konfigurasi (site_config.json), aset publik, dan file pribadi.
  • /logs: Berisi file log untuk debugging (misalnya, web.log, worker.log).
  • /env: Lingkungan virtual Python yang mengisolasi dependensi proyek.

2. Instalasi & Pengaturan Awal

Instalasi Frappe Bench

Proses instalasi manual di Ubuntu 20.04 melibatkan beberapa prasyarat penting. Pastikan semua dependensi ini terinstal dengan benar sebelum menginstal frappe-bench.

  • Dependensi Sistem: git, python3-dev, python3-pip, python3.8-venv
  • Database: MariaDB Server
  • Caching & Antrian: Redis Server
  • JavaScript Runtime: Node.js & Yarn
  • Lainnya: wkhtmltopdf untuk konversi PDF

Inisialisasi Proyek & Perintah Penting

Setelah bench terinstal, Anda dapat memulai proyek baru, membuat situs, dan menginstal aplikasi seperti ERPNext.

# 1. Inisialisasi direktori bench
bench init frappe-bench --frappe-branch version-13

# 2. Pindah ke direktori bench
cd frappe-bench

# 3. Buat situs baru (akan meminta password admin)
bench new-site nama.situs.anda

# 4. Unduh aplikasi ERPNext
bench get-app erpnext --branch version-13

# 5. Instal ERPNext ke situs Anda
bench --site nama.situs.anda install-app erpnext

# 6. Jalankan server pengembangan
bench start

3. Konsep Inti: DocType

3.1 Pembuatan & Tipe Field

DocType adalah cetak biru (blueprint) dari model data di Frappe. Setiap entitas (misalnya, ToDo, Sales Invoice) didefinisikan sebagai DocType, yang kemudian akan membuat tabel di database. Anda dapat membuat DocType baru dari menu "Build".

Frappe menyediakan berbagai tipe field untuk menangani berbagai jenis data, seperti:

  • Data: Teks singkat (string).
  • Link: Referensi ke DocType lain (foreign key).
  • Table: Untuk menyematkan Child DocType.
  • Select: Pilihan dropdown.
  • Date, Currency, Checkbox, Attach Image, dll.

3.2 Properti Field

Setiap field dapat dikustomisasi lebih lanjut dengan properti untuk mengontrol perilakunya di UI. Properti yang paling umum digunakan antara lain:

  • Mandatory: Field harus diisi.
  • Read Only: Field tidak dapat diedit.
  • Hidden: Field tidak ditampilkan di form.
  • Default: Nilai awal untuk field.
  • In List View: Tampilkan field ini di tampilan daftar.
  • Depends On: Tampilkan/sembunyikan field berdasarkan kondisi dari field lain. Contoh: eval:doc.status=="Open"

3.3 Jenis-jenis DocType

Frappe memiliki beberapa jenis DocType untuk kasus penggunaan yang berbeda:

  • Standard: DocType biasa yang dapat dibuat, dibaca, diperbarui, dan dihapus (CRUD).
  • Submittable: Dokumen yang memiliki alur kerja "Draft -> Submitted -> Cancelled". Setelah di-submit, dokumen menjadi read-only. Contoh: Sales Invoice.
  • Single: Hanya ada satu dokumen untuk DocType ini. Berguna untuk menyimpan pengaturan global. Contoh: System Settings.
  • Child Table: DocType yang tidak dapat berdiri sendiri dan harus disematkan di dalam DocType lain menggunakan field tipe "Table".

3.4 Penamaan Dokumen

ID unik setiap dokumen dapat diatur secara otomatis. Opsi penamaan (ditemukan di pengaturan DocType) meliputi:

  • Naming Series: Format penamaan yang paling umum, menggunakan awalan dan nomor urut (misal: INV-.YYYY.-.#####).
  • Berdasarkan Field: Nama dokumen diambil dari nilai salah satu field-nya.
  • Berdasarkan Prompt: Pengguna diminta memasukkan nama secara manual.
  • Format Kustom: Kombinasi teks, field, dan seri (misal: PO-{MM}-{supplier_abbr}-.###).

4. Client-Side Scripting (JavaScript)

4.1 Dasar & Events

Menambahkan interaktivitas dan logika bisnis di sisi browser menggunakan JavaScript. Skrip ditulis dalam file [doctype_name].js yang dibuat secara otomatis. Struktur dasar untuk menangani event adalah blok frappe.ui.form.on. 

frappe.ui.form.on('My DocType', {
    // Event ini berjalan saat form dimuat atau di-refresh
    refresh: function(frm) {
        frappe.msgprint('Form telah di-refresh!');
    },
    // Event ini berjalan saat nilai field 'status' berubah
    status: function(frm) {
        if (frm.doc.status === 'Completed') {
            frappe.show_alert({message: 'Tugas Selesai!', indicator: 'green'});
        }
    },
    // Event validasi sebelum menyimpan
    validate: function(frm) {
        if (frm.doc.age < 18) {
            frappe.throw('Umur tidak boleh kurang dari 18 tahun.');
        }
    }
});

4.2 Manipulasi Form (frm)

Objek frm adalah kunci untuk berinteraksi dengan form saat ini. Anda dapat membaca nilai, mengatur nilai, mengubah properti field, dan banyak lagi.

  • frm.set_value('field', 'value'): Mengatur nilai field dan memicu event on_change.
  • frm.doc.field_name: Mendapatkan nilai field saat ini.
  • frm.set_df_property('field', 'read_only', 1): Mengubah properti field (misal: read_only, hidden, mandatory).
  • frm.add_custom_button(...): Menambahkan tombol kustom di header form.
  • frm.add_child(...): Menambahkan baris baru ke child table.
  • frm.set_intro(...): Menampilkan pesan intro di atas form.

5. Server-Side Scripting (Python)

5.1 Dasar & Hooks

Menjalankan logika bisnis yang kompleks dan aman di sisi server menggunakan Python. Skrip ditulis dalam file [doctype_name].py di dalam sebuah class yang mewarisi dari Document. 

import frappe
from frappe.model.document import Document

class MyDocType(Document):
    # Dijalankan sebelum dokumen disimpan (baik baru atau update)
    def validate(self):
        if self.age < 18:
            frappe.throw("Umur tidak boleh kurang dari 18 tahun.")

    # Dijalankan setelah dokumen di-submit
    def on_submit(self):
        self.status = 'Submitted'
        frappe.msgprint(f"Dokumen {self.name} telah di-submit.")

    # Dijalankan setelah dokumen dibatalkan
    def on_cancel(self):
        self.status = 'Cancelled'

5.2 Metode API Frappe

Frappe menyediakan API yang kaya untuk berinteraksi dengan database dan framework. Ini memungkinkan Anda untuk membuat, membaca, memperbarui, dan menghapus dokumen lain, atau mengambil data secara efisien.

  • frappe.get_doc('DocType', 'name'): Mengambil seluruh objek dokumen.
  • frappe.new_doc('DocType'): Membuat instance dokumen baru.
  • doc.insert() / doc.save(): Menyimpan dokumen ke database.
  • frappe.db.get_value(...): Mengambil satu nilai dari field. Sangat efisien.
  • frappe.db.set_value(...): Mengubah nilai field pada dokumen lain tanpa memuat seluruh dokumen.
  • frappe.db.get_list(...): Mengambil daftar dokumen dengan filter dan field tertentu.

5.3 Memanggil Metode Server dari Client

Anda dapat membuat fungsi Python kustom dan memanggilnya dari JavaScript. Ini sangat berguna untuk menjalankan query yang kompleks atau logika yang harus aman di server. Fungsi Python harus dihiasi dengan @frappe.whitelist(). 

# Di file .py
@frappe.whitelist()
def get_user_age(user_email):
    age = frappe.db.get_value("User", user_email, "birth_date")
    # ... logika perhitungan umur
    return calculated_age

# Di file .js
frappe.call({
    method: "my_app.my_module.doctype.my_doctype.my_doctype.get_user_age",
    args: {
        user_email: frappe.session.user
    },
    callback: function(r) {
        if (r.message) {
            frm.set_value('age', r.message);
        }
    }
});

6. UI & Fitur Lanjutan

Workspace

Kustomisasi halaman utama modul dengan menambahkan pintasan ke DocType, Laporan, atau Halaman, serta kartu tautan untuk navigasi yang lebih baik.

Dashboard

Membuat dasbor visual dengan Number Cards untuk metrik utama dan Dashboard Charts untuk visualisasi data dari DocType atau Laporan.

Frappe Pages

Membuat halaman kustom dengan HTML, CSS, dan JavaScript. Berguna untuk antarmuka yang sangat spesifik yang tidak cocok dengan model form DocType.

Web Forms

Mengekspos DocType sebagai formulir web yang dapat diakses publik atau oleh pengguna yang login. Lengkap dengan skrip klien dan validasi kustom.

7. Pelaporan (Reporting)

Jenis Laporan

Frappe menyediakan sistem pelaporan yang kuat untuk menganalisis data. Anda dapat memilih jenis laporan yang paling sesuai dengan kebutuhan Anda.

  • Report Builder: Cara tercepat membuat laporan. Cukup pilih DocType, lalu seret dan lepas kolom yang Anda inginkan. Cocok untuk laporan sederhana.
  • Query Report: Menulis kueri SQL secara langsung. Memberikan fleksibilitas maksimal untuk mengambil data dari beberapa tabel dan melakukan join.
  • Script Report: Menggunakan Python untuk memproses data. Cocok untuk laporan yang memerlukan kalkulasi kompleks, panggilan API eksternal, atau manipulasi data berat sebelum ditampilkan.

Fitur Laporan Skrip

Pada Laporan Skrip, Anda dapat menambahkan filter interaktif (di file .js), mendefinisikan kolom dan mengambil data (di file .py), serta menambahkan ringkasan dan bagan visual (Pie, Line, Bar) untuk membuat laporan lebih informatif.

8. Format Cetak

Kustomisasi tampilan dokumen saat dicetak atau diunduh sebagai PDF. Penting untuk faktur, surat penawaran, dan dokumen resmi lainnya.

Print Format Builder

Antarmuka visual (drag-and-drop) untuk mengatur tata letak. Anda dapat menambahkan/menghapus field, membuat bagian dan kolom baru, serta menambahkan HTML kustom tanpa perlu coding mendalam.

Format Cetak Kustom (Jinja)

Untuk kontrol penuh atas tampilan, gunakan HTML, CSS, dan Jinja templating. Dengan Jinja, Anda bisa menampilkan data dari field ({{ doc.field_name }}), melakukan perulangan pada tabel anak, menggunakan logika kondisional, dan menerapkan styling kustom dengan CSS.
